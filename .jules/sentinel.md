## 2024-11-20 - Unrestricted File Upload (DoS) and Information Leakage in API
**Vulnerability:** The `/api/compress` route lacked a maximum file size check before reading uploaded files into a buffer (`await file.arrayBuffer()`), which allowed for potential Denial of Service attacks by sending excessively large files. Additionally, the route returned raw error messages (`error.message`) in its 400 responses when `sharp` failed to process images, exposing internal system details or underlying library versions.
**Learning:** Even though Next.js and typical modern server configurations limit request bodies by default, API routes explicitly parsing form data and building internal buffers bypass these if not carefully guarded. Raw exceptions in APIs should never be forwarded directly to clients.
**Prevention:** Always enforce a strict `file.size` check before initiating heavy memory allocations or reading files into buffers. In catch blocks, log detailed errors internally but return only sanitized, generic messages to the client.
