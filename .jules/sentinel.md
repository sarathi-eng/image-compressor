## 2024-05-24 - DoS Risk in Unbounded Image Processing
**Vulnerability:** The API endpoint `app/api/compress/route.ts` was blindly accepting file uploads and allocating unbounded memory buffers for image compression (`Buffer.from(await file.arrayBuffer())`). It lacked size restrictions and MIME type validation, allowing a malicious actor to upload excessively large or non-image files, which could crash the Node.js process (DoS via Out-of-Memory). Additionally, parsing errors leaked internal system or library information to the client.
**Learning:** Any endpoint that processes binary data must enforce strict input size and type limits before allocating memory. Error messages returned to clients must be generic to avoid leaking stack traces or internal mechanics.
**Prevention:** Always validate file sizes (e.g., max 10MB) and verify the `file.type` starts with `"image/"` before calling processing libraries like `sharp`. Catch errors and return generic error messages (e.g., "Image processing failed"), logging the actual error internally.
