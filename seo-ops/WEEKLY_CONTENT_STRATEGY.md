## 1. THE REPEATABLE WEEKLY TEMPLATE SYSTEM

### Template A – Cluster Article (700-1000 words)
*   **H1:** `[PRIMARY_KEYWORD]`: Complete Guide for `[TARGET_AUDIENCE]` in 2026
*   **AEO Definition Block (30-60 words):** Immediately after H1.
    *   *Template:* "`[PRIMARY_KEYWORD]` is `[DIRECT DEFINITION]`. In 2026, understanding `[PRIMARY_KEYWORD]` is crucial for `[TARGET_AUDIENCE]` because it `[BENEFIT/CORE REASON]` while maintaining `[PERFORMANCE/PRIVACY CRITERIA]`."
*   **H2:** Why `[PRIMARY_KEYWORD]` Matters for `[AUDIENCE/USE_CASE]`
*   **H2:** How to `[ACTION]` with `[PRIMARY_KEYWORD]` (Step-by-Step)
    1.  **Step 1:** Select target files.
    2.  **Step 2:** Load the appropriate browser tool.
    3.  **Step 3:** Configure optimization parameters (e.g., quality slider).
    4.  **Step 4:** Execute client-side compression.
    5.  **Step 5:** Download and deploy your optimized assets.
*   **H2:** Top `[NUMBER]` `[STRATEGIES/TOOLS]` for `[PRIMARY_KEYWORD]`
    *   **H3:** `[Strategy/Tool 1]`
    *   **H3:** `[Strategy/Tool 2]`
    *   **H3:** `[Strategy/Tool 3]`
*   **H2:** Common Mistakes to Avoid
    *   *Include comparison/mistakes table:*
        | Mistake | Impact | Fix |
        |---|---|---|
        | `[MISTAKE]` | `[IMPACT]` | `[FIX]` |
*   **AEO Box:** 5 Key Takeaways (Bullet points summarizing the post)
*   **AEO Box:** FAQ Section with 5 questions + FAQPage Schema
*   **Internal Links:** 1 Pillar + 2 Clusters + 1 Tool Page
*   **Schema:** `Article`, `HowTo` (if step-by-step is present), `FAQPage`.

## 3. PROGRAMMATIC CSV MAPPING

### Comparison Pages (Template B)
*   `entity_a` (string) - Name of the first competitor / CompressPro
*   `entity_b` (string) - Name of the second competitor / CompressPro
*   `use_case` (string) - Main workflow compared (e.g. "Web Images", "Batch Video")
*   `feature_list` (text) - JSON list of compared features
*   `pros_a` / `cons_a` (text) - Bullets detailing entity A strengths/weaknesses
*   `pros_b` / `cons_b` (text) - Bullets detailing entity B strengths/weaknesses
*   `verdict` (string) - 1-sentence AI quick summary
*   `internal_link_target` (string) - Target tool URL to drive commercial action

### Use Case/Audience Pages (Template E)
*   `audience` (string) - Target demographic (e.g. "Web Developers", "E-commerce Sellers")
*   `top_file_types` (string) - Common formats utilized by the demographic (e.g. "WebP, JPG")
*   `compression_settings` (string) - Recommended presets (e.g. "75-80% Quality")
*   `workflow_steps` (text) - JSON array of step-by-step instructions
*   `best_practices` (text) - Bullet points detailing top-tier workflow hints
*   `tags` (string) - Categorization tags

### File Format Pages (Template F)
*   `file_format` (string) - e.g. "JPG", "PNG", "WebP", "AVIF", "MP4"
*   `technical_definition` (text) - High fidelity definition text
*   `best_use_case` (string) - Optimal application (e.g. "Detailed Photographs")
*   `compression_tips` (text) - Step-by-step optimization actions
*   `quality_impact` (string) - Expected visual/auditory changes
*   `alternatives` (string) - Related/competing formats
